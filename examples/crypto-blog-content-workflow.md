# Workflow: Создание контента для крипто-блога с помощью AI-агентов

Практическое руководство по использованию агентов The Agency для сайта мониторинга криптовалютных обменников — от анализа аудитории до публикации статьи.

---

## Часть 1. Подобранные агенты и их роли

### 1. Brand Guardian (`design/design-brand-guardian.md`)
**Роль**: Разработка tone of voice и бренд-айдентики блога.

Создаёт фундамент бренда: миссию, ценности, голос, тональность. Для крипто-блога это критически важно — нужно определить, вы звучите как «серьёзный финансовый аналитик», «дружелюбный проводник в мир крипты» или «технический гик». Brand Guardian выдаёт полноценный Brand Voice Guidelines документ с примерами фраз, запрещённой лексикой и вариациями тона для разных ситуаций (новостная статья vs. обучающий гайд vs. обзор обменника).

### 2. Trend Researcher (`product/product-trend-researcher.md`)
**Роль**: Исследование трендов и анализ аудитории крипто-рынка.

Специализируется на выявлении emerging-трендов, конкурентном анализе и оценке возможностей. Использует данные из Google Trends, соцсетей, поисковой аналитики, патентных баз. Умеет делать прогнозы на 3–6 месяцев вперёд. Для крипто-блога: находит горячие темы (какие монеты обсуждают, какие DeFi-протоколы растут, какие обменники набирают популярность), анализирует, что ищет аудитория, и даёт данные для контент-плана.

### 3. SEO Specialist (`marketing/marketing-seo-specialist.md`)
**Роль**: Keyword research и SEO-оптимизация контента.

Работает по принципу «белого SEO»: keyword research, topic clusters, on-page оптимизация, внутренняя перелинковка. Создаёт кластерную структуру контента (pillar pages + supporting articles), анализирует поисковый интент, формирует SEO-брифы для каждой статьи. Для крипто-ниши особенно важен — высокая конкуренция в поисковой выдаче.

### 4. Content Creator (`marketing/marketing-content-creator.md`)
**Роль**: Написание статей и контент-стратегия.

Основной рабочий агент: пишет тексты, создаёт editorial-календарь, адаптирует контент под разные платформы. Умеет писать лонгриды, how-to гайды, обзоры, сравнения. Работает с учётом SEO-требований и бренд-голоса. Для крипто-блога — это тот, кто непосредственно пишет статьи.

### 5. AI Citation Strategist (`marketing/marketing-ai-citation-strategist.md`)
**Роль**: Оптимизация контента для AI-поисковиков.

Новая и крайне актуальная дисциплина. Люди всё чаще спрашивают ChatGPT / Claude / Perplexity «какой лучший обменник крипты» вместо Google. Этот агент аудитирует, цитируют ли AI-системы ваш сайт, анализирует почему конкуренты получают цитирование, и даёт конкретные fix-пакеты: какие страницы создать, какую schema-разметку добавить, как структурировать контент для AI-рекомендаций.

### 6. Narratologist (`academic/academic-narratologist.md`)
**Роль**: Структура и нарратив статей.

Эксперт по сторителлингу и нарративным структурам. Может показаться избыточным для блога, но на самом деле незаменим для превращения сухих обзоров обменников в увлекательное чтение. Помогает выстроить «controlling idea» статьи, создать напряжение, правильно раскрыть тему. Особенно полезен для кейсов, историй о мошенничестве, обучающих лонгридов.

## Часть 2. Практический workflow: поток данных от идеи до публикации

Ниже — пошаговый процесс. Каждый шаг: какой агент, что ему подать на вход, что получить на выходе.

---

### Шаг 0. Фундамент — Tone of Voice (делается один раз)

**Агент**: Brand Guardian

**Что подать на вход**:
```
Я веду блог на сайте мониторинга криптовалютных обменников.

О компании:
- Сайт-агрегатор, который помогает пользователям сравнивать курсы обмена криптовалют
- Аудитория: трейдеры, инвесторы, новички в крипте, люди которые хотят обменять крипту
- Конкуренты: [перечислить 2-3 конкурента]
- Ценности: прозрачность, объективность, защита пользователей от скама

Задача: разработай Brand Voice Guidelines для блога.
Мне нужны:
1. Описание голоса бренда (3-5 характеристик)
2. Вариации тона для разных типов контента (новости, обзоры, гайды, аналитика)
3. Примеры фраз — как мы говорим и как НЕ говорим
4. Правила использования жаргона и терминологии
```

**Что получите на выходе**:
- Документ Brand Voice Guidelines с характеристиками голоса (например: «Экспертный, но доступный», «Объективный, не рекламный», «Защитник пользователя»)
- Таблица тональностей по типам контента
- Список do/don't фраз
- Правила работы с крипто-терминологией

> Этот документ — ваш фундамент. Его вы будете подавать на вход всем остальным агентам как контекст.

---

### Шаг 1. Исследование трендов и формирование контент-плана

**Агент**: Trend Researcher

**Что подать на вход**:
```
Сфера: криптовалюты, обмен криптовалют, мониторинг обменников.

Задачи:
1. Определи 10-15 актуальных тем/трендов в крипто-сфере, которые интересны аудитории
   обменников криптовалют (трейдеры, инвесторы, новички)
2. Для каждой темы оцени:
   - Уровень интереса аудитории (высокий/средний/низкий)
   - Сезонность (привязана ли тема к времени)
   - Конкурентность (много ли контента уже есть)
   - Потенциал для нашего блога (можем ли мы дать уникальную ценность)
3. Проанализируй, что пишут конкуренты: [URL блогов конкурентов]
4. Определи контент-гэпы — темы, которые аудитория ищет, но по которым мало качественного контента

Формат: таблица с приоритизацией тем.
```

**Что получите на выходе**:
- Таблицу из 10-15 тем с оценками по критериям
- Список контент-гэпов
- Рекомендации по приоритетам: что писать первым
- Сигналы о зарождающихся трендах (например: «рост интереса к P2P-обменникам в СНГ на 40%»)

**Пример результата**:

| Тема | Интерес | Конкуренция | Приоритет |
|------|---------|-------------|-----------|
| Как безопасно обменять крипту в 2026 | Высокий | Средняя | P1 |
| Сравнение P2P vs централизованных обменников | Высокий | Низкая | P1 |
| Налогообложение крипты в РФ — изменения | Высокий | Высокая | P2 |
| Как распознать скам-обменник | Средний | Низкая | P1 |

### Шаг 2. SEO-проработка выбранных тем

**Агент**: SEO Specialist

**Что подать на вход** (для каждой темы из Шага 1):
```
Тема статьи: «Как безопасно обменять криптовалюту в 2026 году»
Ниша: мониторинг криптовалютных обменников
Целевая аудитория: трейдеры и инвесторы из СНГ
Сайт: [ваш домен]

Задачи:
1. Подбери ключевые слова: основные + long-tail (с указанием search volume и KD, если возможно оценить)
2. Определи поисковый интент для каждого ключа
3. Проанализируй ТОП-10 выдачи по основному ключу — что там, какой формат контента побеждает
4. Предложи структуру topic cluster: pillar page + supporting articles
5. Составь SEO-бриф для статьи:
   - Title tag и meta description
   - Структура H1-H2-H3
   - Рекомендуемый объём текста
   - Внутренние ссылки (на какие страницы сайта ссылаться)
   - Schema markup (какой тип разметки использовать)
```

**Что получите на выходе**:
- Кластер ключевых слов с volume/KD/intent
- SEO-бриф со структурой статьи, мета-тегами, рекомендациями по объёму
- Стратегию перелинковки
- Рекомендации по schema markup (Article, FAQ, HowTo)

**Пример фрагмента результата**:
```
Pillar Page: «Как обменять криптовалюту — полное руководство 2026»
├── Supporting: «P2P обмен крипты: плюсы и минусы»
├── Supporting: «ТОП-10 признаков скам-обменника»
├── Supporting: «Комиссии обменников: скрытые платежи»
└── Supporting: «KYC в обменниках: зачем нужна верификация»

SEO-бриф для pillar page:
- H1: Как безопасно обменять криптовалюту в 2026 году
- Объём: 3000-4000 слов
- Schema: Article + FAQ
- Target keyword: «обменять криптовалюту»
```

### Шаг 3. Проработка нарратива статьи

**Агент**: Narratologist

**Что подать на вход**:
```
Тип контента: обучающий лонгрид для крипто-блога
Тема: «Как безопасно обменять криптовалюту в 2026 году»
Аудитория: от новичков до опытных трейдеров

SEO-структура (из Шага 2):
[вставить H1-H2-H3 структуру из SEO-брифа]

Задачи:
1. Определи controlling idea статьи — какую главную мысль читатель должен вынести
2. Предложи нарративную структуру: как подать материал, чтобы он увлекал, а не был
   сухим перечислением
3. Где и какие «крючки» использовать, чтобы читатель дочитал до конца
4. Предложи opening hook — первые 2-3 предложения, которые зацепят
5. Где уместны мини-истории, кейсы, примеры из реальной жизни
```

**Что получите на выходе**:
- Controlling idea (например: «Безопасность обмена — это не везение, а набор проверяемых критериев»)
- Нарративная карта статьи: где напряжение растёт, где разрядка, где кульминация
- Opening hook (например: «В марте 2025 года пользователь потерял 2.3 BTC на обменнике, который вчера ещё был в топе рейтинга...»)
- Рекомендации по вставке кейсов и историй
- Карта эмоционального путешествия читателя по статье

### Шаг 4. Написание статьи

**Агент**: Content Creator

**Что подать на вход** (это ключевой момент — сюда стекаются все предыдущие результаты):
```
КОНТЕКСТ БРЕНДА:
[вставить Brand Voice Guidelines из Шага 0]

SEO-БРИФ:
[вставить полный SEO-бриф из Шага 2]

НАРРАТИВНАЯ СТРУКТУРА:
[вставить рекомендации Narratologist из Шага 3]

ДОПОЛНИТЕЛЬНАЯ ФАКТУРА:
- Наш сайт мониторит [X] обменников
- Актуальные данные: средняя комиссия обмена BTC/USDT — [Y]%
- Недавние инциденты: [краткое описание, если есть]

ЗАДАЧА:
Напиши полную статью по теме «Как безопасно обменять криптовалюту в 2026 году»
с учётом всех предоставленных материалов.

Требования:
- Объём: 3000-4000 слов
- Стиль: согласно Brand Voice Guidelines
- SEO: все ключевые слова из брифа интегрированы естественно
- Структура: по нарративной карте
- Включить FAQ-секцию (минимум 5 вопросов) для schema markup
- Включить CTA на использование нашего мониторинга
```

**Что получите на выходе**:
- Готовый текст статьи 3000-4000 слов
- Правильно оформленные заголовки H1-H3
- Интегрированные ключевые слова
- FAQ-секция
- CTA-блоки

### Шаг 5. Оптимизация для AI-поисковиков

**Агент**: AI Citation Strategist

**Что подать на вход**:
```
Бренд: [название вашего сайта]
Домен: [URL]
Категория: мониторинг криптовалютных обменников
Конкуренты: [2-4 конкурента]

Задачи:
1. Сгенерируй 20 промптов, которые пользователь мог бы задать AI-ассистенту
   о сравнении/выборе крипто-обменников
   (например: «какой лучший обменник биткоина», «как выбрать обменник криптовалют»)
2. Проанализируй, какой контент цитируется по этим запросам сейчас
3. Дай рекомендации: какие страницы создать/изменить на нашем сайте, чтобы повысить
   вероятность цитирования
4. Какие schema markup добавить
5. Составь fix pack с приоритизацией
```

**Что получите на выходе**:
- Список из 20 промптов целевой аудитории для AI
- Аудит текущего цитирования (кого цитируют и почему)
- Fix pack: конкретные действия, отсортированные по приоритету
- Рекомендации по schema markup (FAQ, Product, Organization)
- Шаблоны контента, оптимизированного для AI-цитирования

## Часть 4. Как технически использовать файл агента

### Вариант А: Claude Code / Claude в чате

1. Откройте чат с Claude
2. Скопируйте **полное содержимое** файла агента (например, `marketing/marketing-content-creator.md`)
3. Вставьте его в начало промпта как системную инструкцию:

```
[Содержимое файла агента]

---

Теперь, действуя как этот агент, выполни следующую задачу:
[Ваша задача]
```

### Вариант B: Claude Code с project instructions

1. Положите файл агента в папку проекта
2. Добавьте путь к файлу в `CLAUDE.md` или `instructions`
3. Claude Code автоматически подхватит инструкции агента

### Вариант C: Cursor / Windsurf / другие AI-IDE

1. Скопируйте содержимое файла агента в системный промпт или `.cursorrules`
2. Агент будет работать как персонализированный AI-ассистент в IDE

## Часть 5. Минимальный жизнеспособный workflow (быстрый старт)

Если нет времени на полный процесс, вот сокращённая версия из **3 шагов**:

### 1. Brand Voice (один раз) → Brand Guardian
Потратьте 20 минут на создание Brand Voice Guidelines. Это окупится на каждой статье.

### 2. SEO-бриф → SEO Specialist
Для каждой статьи: подайте тему, получите ключевые слова и структуру.

### 3. Написание → Content Creator
Подайте Brand Voice + SEO-бриф + тему. Получите готовый текст.

Три агента, три шага, минимум усилий — максимум результата.

## Рекомендуемая частота использования

| Агент | Когда использовать |
|-------|-------------------|
| Brand Guardian | 1 раз + ревизия раз в полгода |
| Trend Researcher | 1 раз в месяц для обновления контент-плана |
| SEO Specialist | Перед каждой статьёй |
| Narratologist | Для лонгридов и сложных тем |
| Content Creator | Для каждой статьи |
| AI Citation Strategist | 1 раз в квартал для аудита |
