# Примеры и воркфлоу

Этот каталог содержит примеры реальных сценариев использования, демонстрирующие, как агенты The Agency могут работать совместно для решения практических задач.

## Зачем это нужно

Репозиторий agency-agents определяет десятки специализированных агентов в областях инженерии, дизайна, маркетинга, продукта, поддержки, пространственных вычислений и управления проектами. Но определения агентов сами по себе не показывают, что происходит, когда все они работают одновременно над общей задачей. Эти примеры отвечают на вопрос: "Как выглядит совместная работа всего агентства на практике?"

## Nexus Spatial: полное упражнение по продуктовому открытию

**Файл:** `nexus-spatial-discovery.md`

**Тип:** Мультиагентное продуктовое исследование (product discovery) с параллельной работой 8 агентов, выполненное примерно за 10 минут.

**Сценарий:** Веб-исследование выявило возможность на пересечении оркестрации ИИ-агентов и пространственных вычислений (spatial computing). Все агентство было развернуто одновременно для создания продуктовой концепции "Nexus Spatial" -- иммерсивного 3D-центра управления ИИ-агентами для VisionOS и WebXR.

**Задействованные агенты и их результаты:**

| Агент | Роль | Результат |
|-------|------|-----------|
| Product Trend Researcher | Валидация рынка | Анализ рынка ИИ-оркестрации ($13.5B), конкурентный ландшафт (LangChain, CrewAI, n8n, Flowise), реалистичная оценка Vision Pro (~1M устройств), рекомендация стратегии "2D сначала, пространственное потом" |
| Backend Architect | Системная архитектура | 8-сервисная архитектура с движком оркестрации на Rust, 14 таблиц базы данных, реестр типов нод, WebSocket-каналы, архитектура безопасности с mTLS и gVisor |
| Brand Guardian | Бренд-стратегия | Создание новой категории "SpatialAIOps", позиционирование, слоган "Mission Control for the Agent Era", цветовая система, типографика, концепции логотипа |
| Growth Hacker | Стратегия выхода на рынок | Ценообразование (Free/$29/$59/Enterprise), 3-фазный GTM-план, петли роста, open-source стратегия, бюджет первых $50K, целевые метрики |
| Support Responder | Операции поддержки | Трехуровневая структура поддержки с SLA, ИИ-помощник "Nexus Guide" внутри продукта, адаптивный онбординг, Discord-сообщество, программа чемпионов |
| UX Researcher | UX-исследование | 4 персоны пользователей, выявление дебаггинга как ключевого сценария, 7 принципов дизайна, 4-уровневый семантический зум, 16-недельный план исследований |
| Project Shepherd | План выполнения | 35-недельный таймлайн с 6 фазами, 65 тикетов на первые 6 спринтов, 5 команд, топ-5 рисков, бюджет $121K-$155K |
| XR Interface Architect | Пространственный интерфейс | "Command Theater" с изогнутой компоновкой вокруг пользователя, трехслойная система глубины, 3D-граф нод с потоком данных к пользователю, 7 состояний агентов, модель взаимодействия для VisionOS/WebXR/голоса |

**Ключевой вывод:** Все 8 агентов работали параллельно и независимо пришли к согласованным выводам (2D сначала, дебаггинг как главный сценарий, WebXR для охвата), одновременно привнеся уникальную экспертизу каждой области. Результат -- комплексный кросс-функциональный продуктовый план, созданный за одну сессию.

## Воркфлоу: разработка главы книги

**Файл:** `workflow-book-chapter.md`

**Тип:** Одноагентный воркфлоу с итеративными циклами доработки.

**Сценарий:** Автор имеет голосовые заметки, фрагменты и стратегические записи, но ещё не оформленный черновик главы. Задача -- превратить сырой материал в стратегический черновик от первого лица с явными точками для доработки.

**Задействованный агент:** Book Co-Author -- конвертирует исходный материал в версионированный черновик главы с редакторскими заметками и вопросами для следующего шага.

**Процесс:** Агент получает цель книги, целевую аудиторию, тему главы, сырой материал (голосовые заметки, записи, фрагменты историй, позиционирование) и выдает: (1) цель главы и её стратегическую роль в книге, (2) уточняющие вопросы, (3) полный черновик главы, (4) редакторские заметки с предположениями и пробелами в доказательствах, (5) конкретные запросы на следующую итерацию доработки.

**Критерии качества:** Черновик сохраняет голос автора от первого лица, имеет одно ясное обещание и внутреннюю логику, утверждения привязаны к исходному материалу или отмечены как допущения, мотивационные штампы удалены.

## Воркфлоу: спринт по лендинг-странице

**Файл:** `workflow-landing-page.md`

**Тип:** Мультиагентный воркфлоу для создания конверсионно-оптимизированной лендинг-страницы за один день.

**Сценарий:** Необходимо запустить лендинг-страницу для нового продукта "FlowSync" (платформа API-интеграций) к концу дня. Страница должна выглядеть профессионально, конвертировать посетителей и быть готова к деплою.

**Задействованные агенты (4):**

| Агент | Роль | Этап |
|-------|------|------|
| Content Creator | Написание копирайтинга (герой, проблемы, этапы, социальное доказательство, ценообразование, CTA) | Утро (параллельно с дизайном) |
| UI Designer | Дизайн-спецификации (вайрфрейм, цветовая палитра, типографика, компоненты, брейкпоинты) | Утро (параллельно с копирайтингом) |
| Frontend Developer | Сборка HTML + Tailwind CSS (адаптивная, быстрая, доступная, с формой подписки) | Середина дня |
| Growth Hacker | Оптимизация конверсии (CTA выше фолда, ясность ценностного предложения, A/B-тесты, SEO) | После обеда |

**Ключевые паттерны:** параллельный запуск независимых задач (копирайтинг и дизайн), точка слияния (фронтенд-разработчик ждет оба результата), петля обратной связи (Growth Hacker проверяет, Frontend Developer вносит правки), таймбоксинг каждого этапа для предотвращения расползания скоупа.

## Воркфлоу: стартап MVP

**Файл:** `workflow-startup-mvp.md`

**Тип:** Мультиагентный воркфлоу для создания MVP за 4 недели -- от идеи до запуска.

**Сценарий:** Создание SaaS-продукта "RetroBoard" -- инструмента для ретроспектив удаленных команд. За 4 недели необходимо выпустить рабочий продукт с регистрацией пользователей, основной функциональностью и лендинг-страницей.

**Задействованные агенты (7):**

| Агент | Роль |
|-------|------|
| Sprint Prioritizer | Разбивка проекта на 4 недельных спринта с критериями приемки |
| UX Researcher | Быстрый конкурентный анализ (EasyRetro, Retrium, Parabol), выявление дифференциатора |
| Backend Architect | Проектирование API, схемы базы данных (PostgreSQL), WebSocket-событий, стратегии аутентификации |
| Frontend Developer | Создание React-приложения (TypeScript, Tailwind, Socket.io-client) |
| Rapid Prototyper | Быстрое прототипирование первой рабочей версии |
| Growth Hacker | Планирование запуска (Product Hunt, Reddit, Hacker News, Twitter) |
| Reality Checker | Контрольные проверки на середине проекта и перед запуском (GO/NO-GO) |

**Понедельный план:** Неделя 1 -- исследование и архитектура (Sprint Prioritizer + UX Researcher параллельно, затем Backend Architect). Неделя 2 -- разработка основных функций (Frontend Developer + Rapid Prototyper, затем промежуточная проверка Reality Checker). Неделя 3 -- полировка + лендинг (Frontend Developer продолжает, Growth Hacker начинает план запуска). Неделя 4 -- запуск (финальная проверка Reality Checker с решением GO/NO-GO).

**Ключевые паттерны:** последовательная передача (выход одного агента -- вход следующего), параллельная работа (UX Researcher и Sprint Prioritizer одновременно на неделе 1), контрольные точки качества (Reality Checker на середине и перед запуском), передача контекста через копирование полного вывода агентов.

## Воркфлоу: стартап MVP с персистентной памятью

**Файл:** `workflow-with-memory.md`

**Тип:** Тот же воркфлоу стартап MVP, но с MCP-сервером памяти для автоматической передачи состояния между агентами.

**Проблема ручных передач:** В стандартном воркфлоу пользователь вручную копирует выход каждого агента и вставляет его в промпт следующего. Это работает для небольших проектов, но ломается при тайм-аутах сессий, когда нескольким агентам нужен один и тот же контекст, при откате после неудачной проверки качества и при многодневных проектах.

**Решение:** С MCP-сервером памяти агенты автоматически сохраняют свои результаты в память и извлекают нужный контекст. Передача упрощается до: "Project: RetroBoard. Recall previous context and design the API" -- агент сам находит в памяти план спринтов и исследовательский бриф.

**Ключевые отличия от стандартного воркфлоу:**

| Аспект | Стандартный | С памятью |
|--------|-----------|-----------|
| Передачи | Копирование полного вывода между агентами | Агенты автоматически извлекают нужное из памяти |
| Потеря контекста | Тайм-аут сессии = потеря всего | Память сохраняется между сессиями |
| Контекст от нескольких агентов | Ручная компиляция | Агент ищет по тегу проекта |
| Восстановление после ошибки QA | Ручное описание проблемы | Агент вспоминает отзыв + откатывается |
| Многодневные проекты | Восстановление контекста каждую сессию | Агент продолжает с того места, где остановился |

**Ключевые паттерны:** теги проекта на каждой записи в памяти, теги результатов для принимающего агента, полная видимость для Reality Checker через память, откат вместо ручной отмены.
