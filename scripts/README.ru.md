# Скрипты и утилиты

Этот каталог содержит служебные скрипты для конвертации, установки и валидации агентов The Agency.

---

## convert.sh -- Конвертация агентов в форматы инструментов

**Назначение:** Считывает все файлы агентов из стандартных директорий категорий (academic, design, engineering, game-development, marketing, paid-media, sales, product, project-management, testing, support, spatial-computing, specialized) и создает конвертированные файлы в `integrations/<tool>/`.

**Использование:**

```bash
./scripts/convert.sh [--tool <name>] [--out <dir>] [--parallel] [--jobs N] [--help]
```

**Поддерживаемые инструменты:**

| Инструмент | Формат вывода |
|------------|---------------|
| `antigravity` | Отдельный `SKILL.md` для каждого агента с фронтматтером Antigravity (name, description, risk, source, date_added) в `integrations/antigravity/agency-<slug>/` |
| `gemini-cli` | Файлы `SKILL.md` с минимальным фронтматтером + манифест `gemini-extension.json` в `integrations/gemini-cli/skills/<slug>/` |
| `opencode` | Файлы `.md` с фронтматтером OpenCode (name, description, mode: subagent, color в HEX) в `integrations/opencode/agents/` |
| `cursor` | Файлы правил `.mdc` с фронтматтером Cursor (description, globs, alwaysApply) в `integrations/cursor/rules/` |
| `openclaw` | Рабочие пространства из трех файлов (SOUL.md, AGENTS.md, IDENTITY.md) с автоматическим разделением секций по ключевым словам в заголовках |
| `qwen` | Файлы `.md` для Qwen Code SubAgent с фронтматтером (name, description, tools) в `integrations/qwen/agents/` |
| `aider` | Единый файл `CONVENTIONS.md`, объединяющий всех агентов с разделителями |
| `windsurf` | Единый файл `.windsurfrules`, объединяющий всех агентов с визуальными разделителями |
| `all` | Все инструменты (по умолчанию) |

**Основные возможности:**

- Извлечение YAML-фронтматтера (name, description, color, emoji, vibe) и тела каждого агента
- Преобразование имен агентов в kebab-case slug (например, "Frontend Developer" -> "frontend-developer")
- Преобразование именованных цветов в HEX-коды для OpenCode (cyan -> #00FFFF, blue -> #3498DB и т.д.)
- Интеллектуальное разделение секций для OpenClaw: секции identity/communication/style/rules попадают в SOUL.md, остальные -- в AGENTS.md
- Прогресс-бар в стиле tqdm для отслеживания хода конвертации
- Поддержка параллельного выполнения (`--parallel`) с буферизацией вывода для каждого инструмента
- Настраиваемое количество параллельных задач (`--jobs N`, по умолчанию -- количество ядер CPU)
- Цветной вывод в терминале (отключается при отсутствии TTY или установленной переменной NO_COLOR)

**Важно:** Скрипт никогда не изменяет пользовательские конфигурационные директории -- для этого используется `install.sh`.

## install.sh -- Установка агентов в локальные инструменты

**Назначение:** Считывает конвертированные файлы из `integrations/` и копирует их в соответствующие конфигурационные директории для каждого инструмента.

**Использование:**

```bash
./scripts/install.sh [--tool <name>] [--interactive] [--no-interactive] [--parallel] [--jobs N] [--help]
```

**Способы установки для каждого инструмента:**

| Инструмент | Директория назначения | Область |
|------------|----------------------|---------|
| `claude-code` | `~/.claude/agents/` | Глобальная (пользовательская) |
| `copilot` | `~/.github/agents/` + `~/.copilot/agents/` | Глобальная (пользовательская) |
| `antigravity` | `~/.gemini/antigravity/skills/` | Глобальная (пользовательская) |
| `gemini-cli` | `~/.gemini/extensions/agency-agents/` | Глобальная (пользовательская) |
| `openclaw` | `~/.openclaw/agency-agents/` | Глобальная (пользовательская) |
| `qwen` | `.qwen/agents/` (в текущей директории) | Привязана к проекту |
| `opencode` | `.opencode/agents/` (в текущей директории) | Привязана к проекту |
| `cursor` | `.cursor/rules/` (в текущей директории) | Привязана к проекту |
| `aider` | `CONVENTIONS.md` (в текущей директории) | Привязана к проекту |
| `windsurf` | `.windsurfrules` (в текущей директории) | Привязана к проекту |

**Основные возможности:**

- **Автоматическое обнаружение инструментов:** При запуске без `--tool` скрипт сканирует систему на наличие установленных инструментов (проверяет наличие CLI-команд и конфигурационных директорий) и устанавливает агентов только для обнаруженных.
- **Интерактивный режим:** При запуске в терминале без `--tool` показывается интерактивный селектор с чекбоксами, где обнаруженные инструменты предварительно отмечены. Поддерживается переключение отдельных инструментов по номеру, массовый выбор (all/none/detected).
- **Параллельная установка:** Флаг `--parallel` запускает установку для каждого инструмента параллельно с буферизацией вывода.
- **Визуальный интерфейс:** ASCII-рамки для заголовков, прогресс-бар, цветовая индикация статусов.
- **Защита существующих файлов:** Для Aider и Windsurf скрипт не перезаписывает существующие файлы (`CONVENTIONS.md`, `.windsurfrules`), предупреждая пользователя.
- **Регистрация в OpenClaw:** При наличии CLI OpenClaw агенты автоматически регистрируются через `openclaw agents add`.
- **Кроссплатформенность:** Поддерживается Linux, macOS (bash 3.2+), Windows Git Bash / WSL.

## lint-agents.sh -- Валидация файлов агентов

**Назначение:** Проверяет корректность markdown-файлов агентов, выявляя ошибки в структуре и отсутствующие рекомендуемые секции.

**Использование:**

```bash
./scripts/lint-agents.sh [file ...]
```

Если файлы не указаны, скрипт сканирует все стандартные директории агентов: design, engineering, game-development, marketing, paid-media, product, project-management, testing, support, spatial-computing, specialized.

**Выполняемые проверки:**

| Тип | Проверка | Уровень |
|-----|----------|---------|
| Фронтматтер | Наличие открывающих `---` | ERROR |
| Фронтматтер | Непустой блок фронтматтера | ERROR |
| Фронтматтер | Наличие обязательных полей: `name`, `description`, `color` | ERROR |
| Секции | Наличие рекомендуемых секций: "Identity", "Core Mission", "Critical Rules" | WARN |
| Содержимое | Тело файла содержит не менее 50 слов | WARN |

**Результат:** Скрипт выводит общее количество ошибок и предупреждений. При наличии хотя бы одной ошибки (ERROR) возвращает код выхода 1 (FAILED), иначе -- 0 (PASSED). Это позволяет использовать скрипт в CI/CD-пайплайне как проверку перед мержем.
