# Интеграции с инструментами

Этот каталог содержит интеграции The Agency и конвертированные форматы агентов для поддерживаемых инструментов разработки на базе ИИ.

## Поддерживаемые инструменты

- **[Claude Code](#claude-code)** -- агенты в формате `.md`, используются напрямую без конвертации
- **[GitHub Copilot](#github-copilot)** -- агенты в формате `.md`, используются напрямую без конвертации
- **[Antigravity](#antigravity)** -- файлы `SKILL.md` для каждого агента в `antigravity/`
- **[Gemini CLI](#gemini-cli)** -- расширение + файлы `SKILL.md` в `gemini-cli/`
- **[OpenCode](#opencode)** -- файлы агентов `.md` в `opencode/`
- **[OpenClaw](#openclaw)** -- рабочие пространства `SOUL.md` + `AGENTS.md` + `IDENTITY.md`
- **[Cursor](#cursor)** -- файлы правил `.mdc` в `cursor/`
- **[Aider](#aider)** -- единый файл `CONVENTIONS.md` в `aider/`
- **[Windsurf](#windsurf)** -- единый файл `.windsurfrules` в `windsurf/`
- **[MCP Memory](#mcp-memory)** -- интеграция с MCP-сервером памяти для персистентного хранения контекста

## Быстрая установка

```bash
# Установить для всех обнаруженных инструментов автоматически
./scripts/install.sh

# Установить для конкретного инструмента
./scripts/install.sh --tool antigravity
./scripts/install.sh --tool copilot
./scripts/install.sh --tool openclaw
./scripts/install.sh --tool claude-code

# Для Gemini CLI сначала нужно сгенерировать файлы интеграции
./scripts/convert.sh --tool gemini-cli
./scripts/install.sh --tool gemini-cli
```

Для инструментов с привязкой к проекту (OpenCode, Cursor, Aider, Windsurf) установщик следует запускать из корневой директории целевого проекта.

## Перегенерация файлов интеграции

При добавлении или изменении агентов все файлы интеграции можно перегенерировать:

```bash
./scripts/convert.sh
```

## Claude Code

The Agency изначально создавалась для Claude Code. Агенты работают нативно без какой-либо конвертации -- используется стандартный формат `.md` с YAML-фронтматтером. Файлы агентов копируются в директорию `~/.claude/agents/`. Для активации агента достаточно обратиться к нему по имени в сессии Claude Code, например: "Activate Frontend Developer". Установка выполняется командой `./scripts/install.sh --tool claude-code` или ручным копированием `.md` файлов из нужной категории.

---

## GitHub Copilot

GitHub Copilot также работает с агентами нативно, без необходимости конвертации. Агенты используют тот же формат `.md` с YAML-фронтматтером. Файлы копируются сразу в две директории: `~/.github/agents/` и `~/.copilot/agents/`. Активация агента происходит по имени в сессии Copilot. Установка выполняется командой `./scripts/install.sh --tool copilot`.

## Antigravity

Все 61 агент The Agency устанавливаются как навыки (skills) Antigravity. Каждый агент конвертируется в отдельный файл `SKILL.md` и получает префикс `agency-` во избежание конфликтов с существующими навыками (например, `agency-frontend-developer`, `agency-backend-architect`). Навыки устанавливаются в директорию `~/.gemini/antigravity/skills/`. Файлы содержат специальный фронтматтер, совместимый с Antigravity, включая поля `name`, `description`, `risk`, `source` и `date_added`. Для активации навыка в Antigravity используется его slug-имя.

## Gemini CLI

Агенты упаковываются как расширение (extension) Gemini CLI с отдельными файлами навыков для каждого агента. Расширение устанавливается в `~/.gemini/extensions/agency-agents/` и включает манифест `gemini-extension.json` и директорию `skills/`, где каждый агент представлен в виде `SKILL.md` с минимальным фронтматтером (только `name` и `description`). Поскольку манифест и файлы навыков являются генерируемыми артефактами, перед установкой необходимо выполнить `./scripts/convert.sh --tool gemini-cli`. После этого расширение устанавливается командой `./scripts/install.sh --tool gemini-cli`.

## OpenCode

Агенты OpenCode -- это файлы `.md` с YAML-фронтматтером, хранящиеся в `.opencode/agents/`. Конвертер автоматически преобразует именованные цвета в HEX-коды и добавляет `mode: subagent`, чтобы агенты вызывались по запросу через `@agent-name`, не загромождая основной список агентов. Агенты привязаны к проекту (project-scoped), поэтому установщик нужно запускать из корня проекта. Для глобальной доступности файлы можно скопировать в `~/.config/opencode/agents/`. Активация агента в OpenCode осуществляется через префикс `@`, например: `@frontend-developer help build this component`.

## OpenClaw

Каждый агент конвертируется в рабочее пространство (workspace) OpenClaw, содержащее три файла: `SOUL.md` (персона, тон, границы поведения), `AGENTS.md` (миссия, результаты, рабочий процесс) и `IDENTITY.md` (эмодзи, имя, описание из фронтматтера). Разделение на SOUL и AGENTS происходит автоматически на основе ключевых слов в заголовках разделов: секции, связанные с идентичностью, стилем общения и правилами, попадают в SOUL.md, остальные -- в AGENTS.md. Рабочие пространства устанавливаются в `~/.openclaw/agency-agents/` и регистрируются через CLI OpenClaw (если он доступен). После установки требуется перезапуск шлюза: `openclaw gateway restart`.

## Cursor

Все 61 агент конвертируются в файлы правил `.mdc` для Cursor. Правила привязаны к проекту -- установщик создает файлы в `.cursor/rules/<agent-slug>.mdc` в текущей директории проекта. Каждый файл содержит фронтматтер с полями `description`, `globs` (пустое по умолчанию) и `alwaysApply: false`. Для активации правила в Cursor можно обратиться к агенту через `@frontend-developer` или сделать правило постоянно активным, изменив `alwaysApply` на `true` и указав glob-паттерн файлов.

## Aider

Все 61 агент объединяются в единый файл `CONVENTIONS.md`, который Aider автоматически считывает при наличии в корне проекта. Файл содержит заголовок с пояснением, что это агенты The Agency, и список всех агентов с их описаниями и телом инструкций, разделенных горизонтальными линиями. Для активации агента в сессии Aider достаточно сослаться на него по имени, например: "Use the Frontend Developer agent to refactor this component". Файл также можно передать напрямую: `aider --read CONVENTIONS.md`.

---

## Windsurf

Все 61 агент объединяются в единый файл `.windsurfrules`, который размещается в корне проекта. Формат аналогичен Aider -- каждый агент представлен секцией с именем, описанием и полным текстом инструкций, разделенных линиями из знаков `=`. Правила привязаны к проекту. Для активации агента в Windsurf достаточно сослаться на него по имени в промпте.

## MCP Memory

Интеграция с MCP-сервером памяти (Model Context Protocol) позволяет наделить любого агента персистентной памятью между сессиями. По умолчанию агенты The Agency начинают каждую сессию с чистого листа, а контекст передается вручную через копирование. MCP Memory решает эту проблему, предоставляя три ключевые возможности:

- **Кросс-сессионная память**: агент помнит решения, результаты и контекст из предыдущих сессий.
- **Непрерывность передачи**: при передаче задачи между агентами принимающий агент автоматически извлекает контекст из памяти без ручного копирования.
- **Откат при ошибке**: при провале проверки качества или ошибочном решении можно откатиться к последнему рабочему состоянию.

Интеграция требует MCP-сервера с поддержкой инструментов `remember`, `recall`, `rollback` и `search`. В каталоге также содержится скрипт `setup.sh`, который помогает настроить MCP-сервер памяти (проверяет наличие конфигурации MCP-клиента в стандартных местах), и пример агента `backend-architect-with-memory.md` -- стандартный Backend Architect с добавленной секцией Memory Integration. Для добавления памяти к любому агенту достаточно добавить секцию "Memory Integration" в его промпт с инструкциями по использованию `remember`, `recall` и `rollback` в ключевые моменты работы.
